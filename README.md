# 学生备忘录

## 1 项目概述

大学阶段是人们真正步入社会的过渡期，大学生在学习知识之外，更重要的是学会如何应对来自于外界的信息，学会如何调整自己的生活节奏或生活方式，让自己能够更快地适应新的生活环境，面对新的挑战。大学生们日常会接触到各种各样的活动宣传，不定时的课程调整通知，纷繁庞杂的新鲜知识，各型各色的娱乐方式等。现如今互联网生态日趋成熟，对于当代大学生来说，每天需要从来自于网络的海量信息中，筛选出具备较高的价值的，能够为自己提供帮助的内容，是比较困难的。与此同时，还面会临着如何规划好课程课业、课外学习、娱乐放松、人际交往、职业发展等方面的安排的压力。为了帮助大学生克服这些困难和挑战，实验开发学生备忘录系统。

## 2 项目目标

学生备忘录**旨在**帮助大学生们记录下自己的各种想法、各种目标，以实现更合理的自我调整和规划，学会如何分清主次，做好取舍，帮助大学生成长。

**系统的开发原则**，具体为操作简单，上手容易，用户友好，营造出较为放松的环境，保证较为舒适的使用体验，保证界面美观。

## 3 需求分析

### 3.1 外部接口需求

#### 3.1.1 用户接口

（1）普通用户登录页

- 信息区：输入框填写用户名和密码
- 功能区：登录按钮提交用户名和密码
- 链接区：链接跳转到注册页或管理员登录页
- 验证区：验证用户名和密码是否符合规范或是否正确
- 展示区：页面内容的标题“登录”和网页的标题“Login To STM”

（2）管理员登录页

- 信息区：输入框填写管理员用户名和密码
- 功能区：登录按钮提交管理员用户名和密码
- 链接区：链接跳转到普通用户登录页
- 验证区：验证用户类型是否为管理员
- 展示区：页面内容的标题“Admin登录”和网页的标题“Admin Login To STM”

（2）注册页

- 信息区：输入框填写用户名、年龄、邮箱、密码、确认密码，单选按钮选择性别
- 功能区：注册按钮提交用户名、性别、年龄、邮箱、密码
- 链接区：链接跳转到普通用户登录页
- 验证区：验证用户名、年龄、邮箱、密码、确认密码是否符合规范
- 展示区：页面内容的标题“注册”和网页的标题“Register For STM”

（3）普通用户主页

- 展示区：页面内容的标题“Welcome To STM”、网页的标题“Welcome To STM”、当前时间格式为“YYYY年MM月DD日 星期X”
- 功能区：“查看我的事件”按钮跳转到用户事件页，“查看我的日记”按钮跳转到用户日记页

（4）管理员主页

- 展示区：页面内容的标题“Admin”、网页的标题“Admin”、当前时间格式为“YYYY年MM月DD日 星期X”
- 功能区：“管理事件”按钮跳转到事件管理页，“管理日记”按钮跳转到日记管理页，“管理用户”按钮跳转到用户管理页（未完成）

（5）用户事件页

- 展示区：页面内容的标题“My DDLS”、网页的标题“Create My DDLS”、展示用户事件的卡片
- 功能区：添加按钮弹出内置关闭和添加按钮的“添加模态框”、“编辑内容”按钮弹出内置关闭和保存按钮的“编辑模态框”、“删除内容”按钮删除用户事件卡片
- 信息区：“添加模态框”中的输入框填写事件截止日期和事件内容、“编辑模态框”中的输入框修改事件的截止日期和事件内容
- 验证区：验证两个模态框中的日期和内容是否符合规范、验证用户是否确认删除事件卡片

（6）用户日记页

- 展示区：页面内容的标题“My Diaries”、网页的标题“Create My Diaries”、展示用户日记的卡片
- 功能区：添加按钮弹出内置关闭和添加按钮的“添加模态框”、“修改内容”按钮弹出内置关闭和保存按钮的“编辑模态框”、“删除日记”按钮删除用户日记卡片
- 信息区：“添加模态框”中的输入框填写日记日期、日记内容、心情内容，“编辑模态框”中的输入框修改日记内容和心情内容。
- 验证区：验证两个模态框中的内容是否符合规范、验证用户是否确认删除日记卡片

### 3.2 系统需求

#### 3.2.1 页面设计

主题颜色：淡黄棕色。不会产生压力，使精神放松。

背景添加几何元素及动效：使思维活跃，产生灵感。

#### 3.2.2 用户登录

用户名：4-16 位的英文字符、数字、下划线

密码：6-16 位的英文字符、数字、下划线

#### 3.2.3 用户注册

用户名：4-16 位的英文字符、数字、下划线

年龄：0-150 之间的数值

邮箱：QQ邮箱、163邮箱、126邮箱、Gmail、矿大邮箱

密码：6-16 位的英文字符、数字、下划线

确认密码：和密码保持一致

#### 3.2.4 事件记录

年份：2022-2050 之间的数值

月份：01-12 之间的数值（两位，缺位补0）

日期：根据平年或闰年，以及相应的月份计算出合法日期（两位，缺位补０）

事件内容：不能为空

#### 3.2.5 日记记录

年份：2022-2050 之间的数值

月份：01-12 之间的数值（两位，缺位补0）

日期：合法的日期（两位，缺位补０）

日记内容：不能为空

心情内容：不能为空

## 4 总体设计

### 4.1 系统功能概述

- 管理员可在管理员登陆页或普通用户登录页，通过“用户名：admin，密码：333333”登录进入系统。
- 未注册的普通用户，点击普通用户登录页的“去注册”链接，跳转到注册页进行注册。
- 普通用户注册完成后，在普通用户登录页，可通过正确的用户名和密码登录进入学生备忘录系统。
- 在普通用户主页，显示欢迎语和当天日期，用户可选择查看自己的事件记录和日记记录。
- 在用户事件页，当前登录用户可查看已有的待完成事件（Deadlines），可以点击页面右上角的添加按钮，添加待完成的事件，设置截止日期并编辑事件内容，可以点击事件卡片上浮现的编辑内容按钮，编辑相应的事件和截止时间，可以点击删除记录按钮，删除相应的事件。
- 在用户日记页，当前登录用户可查看自己已有的日记记录（Diaries），可以点击页面右上角的添加按钮，写下新的日记，编辑日记内容和心情内容，可以点击日记卡片上浮现的修改内容按钮，编辑相应的日记卡片内容，可以点击删除日记按钮，删除相应的日记。

### 5.3 事件模块

事件编辑页（以卡片形式展示，按时间的临近程度顺序排列，时间精确到天，卡片上展示截止时间、事件内容，有`编辑内容`链接和`删除记录`按钮）。点击网页右上角的添加按钮，弹出交互式窗口，读取用户输入的截止时间、事件内容，交互式窗口右下角有关闭和添加按钮，用户点击关闭按钮，退出交互式窗口，系统不进行任何操作，用户点击添加按钮，系统将用户新建的事件添加到数据库中。点击卡片上的`编辑内容`按钮，弹出交互式窗口，允许用户修改截止时间和记录内容，交互式窗口右下角有关闭和保存按钮，用户点击关闭按钮，退出交互式窗口，系统不进行任何操作，用户点击保存按钮，系统根据事件相应的id，修改数据库中相应的事件信息。点击卡片上的`删除记录`按钮，浏览器弹出请用户确认的弹窗，待用户确认后，删除相应的事件。

![事件模块活动图](E:\myprojects\javawebexpr0304-studentmemo\png\事件模块活动图.png)

### 5.4 日记模块

日记编辑页（以卡片形式展示，时间精确到天，卡片上显示日期、日记内容、心情内容，有`修改内容`和`删除日记`按钮）。点击网页右上角的添加按钮，弹出交互式窗口，读取用户输入的当天时间、日记内容、心情内容，交互式窗口右下角有关闭和添加按钮，功能类似事件编辑页中相应的按钮。点击卡片上的`修改内容`按钮，弹出交互式窗口，允许用户修改日记内容和心情内容，交互式窗口右下角有关闭和保存按钮，功能类似事件编辑页中相应的按钮。点击卡片上的`删除记录`按钮，浏览器弹出请用户确认的弹窗，待用户确认后，删除相应的日记。
